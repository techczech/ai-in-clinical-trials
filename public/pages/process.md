# How this Research was Conducted

This page provides a narrative of the tools and methods used during this project on 28 January 2026. This description was generated by Gemini 3 Flash based on the [original process notes](https://github.com/dominiklukes/ai-in-clinical-trials/blob/main/processnotes/ProcessNotes.md) documented on GitHub.

## Analytical and Deep Research Tools

A variety of "semantic machines" were used to gather evidence and case studies regarding generative AI in clinical trials. These tools were not used in a strict linear sequence but rather as complementary methods for deep research.

- **Consensus**: Used to address the broad feasibility question: "How useful is generative AI in the context of clinical trials?" This resulted in a [synthesis of research findings](https://consensus.app/search/generative-ai-clinical-trials/FG_gpejsRui5nJ7Lp0kLQw/) highlighting early-stage evidence and high-potential areas.
- **Elicit**: Employed through its research report workflow with the prompt "Survey of the use of Large Language Models in clinical trials." This created a [structured literature review](https://elicit.com/review/5538287b-d2b2-45c7-b29b-84b9462fece2) and an exported report found in the project's source files.
- **ChatGPT Deep Research**: Utilized to surface reported case studies from across the internet, including social media, blogs, news, and publications. The model was specifically instructed to only describe reported information without making assumptions. A [shared result](https://chatgpt.com/s/dr_697b1269229c8191a6a1daf93dbb85ea) documents this discovery process.
- **NotebookLM**: Functioned as another Deep Research tool to process source materials in a [shared notebook](https://notebooklm.google.com/notebook/5437f419-a79b-4088-b808-84ab44e48b01). It generated several illustrative outputs, including a CSV table of findings and dates (using the prompt "Create a table of dates of report and the finding with a column of positive and negative lessons learned"), a mindmap, a video overview, and a briefing document.

## Scenario Development and Role Modeling

To ground the research in the practical context of the **Surgical Intervention Trials Unit (SITU) at Oxford**, **ChatGPT (GPT-5.2 Thinking)** was used to generate descriptive vignettes. These scenarios illustrate how various roles—such as trial managers, principal investigators, research physiotherapists, and DPhil students—might perform tasks in a future Oxford environment. A follow-up prompt ("for each scenario write a 2-3 vignette of what actually happens - not terse but descriptive for someone like me") was used to ensure the outputs were detailed enough to power the searchable scenario browser on this site.

## Website Architecture and Build

The construction of this website was performed by **Antigravity**. The research journey began with a dictated prompt via **MacWhisper**, which the Antigravity agent then executed using **Gemini 3 Flash**. 

The process involved:
- Initializing the repository and performing an initial git commit of all source files and process notes.
- Designing the website to function as both a research presentation and a process illustration.
- Implementing a searchable and browsable interface for the research results and clinical scenarios.
- Refactoring the content into a Markdown-based structure (storing pages in a `pages` folder) to allow for easier content updates and maintenance.

The project is hosted on **Cloudflare** with all source files and research materials maintained on **GitHub**.
